# Introduction to Version Control

### 1. What is Version Control?

Version Control is a system that manages different versions of files such as code and documents.

It keeps track of changes over time and allows you to:

- Collaborate with others
- View the history of changes
- Restore older versions if needed

### 2. Understanding "Versions"

A version is simply a new state of something after changes.

**Examples:**

**Books:**
- Book v1 → Original edition
- Book v2 → Added new chapters

**Applications:**
- WhatsApp v2.20 → Basic features
- WhatsApp v2.21 → Added voice calls
- WhatsApp v2.22 → Bug fixes

### 3. Why Version Control?

- Multiple developers can work together
- Prevents overwriting others' code
- Stores change history
- Can restore old versions
- Keeps source code safe

### 4. What is Git?

**Git** is a Version Control System (VCS).

It is also called a Source Code Management System (SCM).

With Git, developers can:

- Track changes in code
- Create and manage multiple versions
- Work collaboratively without disturbing others' code

### 5. Example Scenario

Imagine a team of 10 developers building an online shopping application.

Without version control, developers may overwrite each other's code and important files can be lost. Multiple people working on the same files can create confusion and errors.

With **Git**, each developer can work on separate branches. Git tracks every change and stores version history. If mistakes happen, older versions can be restored easily.

This makes collaboration safe and organized.
