# Why Do We Need Git?

### 1. Git Provides Version Control

Git helps developers track and manage changes made in source code. It records every modification as a new version so that previous versions can be restored when needed.

**Example:**
If you accidentally remove some code from a project, Git allows you to return to an older version.

🔑 **Benefit:** Helps recover code and prevents loss of important work.

---

### 2. Git Supports Branching

Git allows developers to create separate branches for different tasks or features. Changes made in one branch do not affect the main project.

**Example:**
A developer creates a branch called `login-feature` to build a login page.

 **Benefit:** Enables safe development without affecting the original code.

---

### 3. Git Allows Merging

Git can combine changes from multiple branches into a single branch after development is completed.

**Example:**
After completing the login page in `login-feature`, it can be merged into the `main` branch.

 **Benefit:** Helps integrate different features into one project.

---

### 4. Git Enables Collaboration

Git allows multiple developers to work together on the same project simultaneously.

**Example:**
Developer A works on the login page, Developer B works on the payment page, and Developer C works on the dashboard.

 **Benefit:** Improves teamwork and avoids overwriting others’ work.
